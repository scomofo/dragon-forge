# Navigation and Service Ticket UI

## Purpose

Dragon Forge should not present sidequests as a traditional quest log.

Skye is becoming an administrator, so optional work should appear as Service Tickets: system errors, regional anomalies, restore requests, and corruption alerts.

## Navigation UI

The navigation HUD should communicate:

- Direction to the Hardware Husk.
- Direction to the active waypoint.
- Sector stability.
- Packet velocity toward the selected target.

The compass is not just north. It is a system health instrument.

## Vector Compass

Current Godot direction:

- Use a custom `Control` HUD ring.
- Draw a cyan Husk vector from Skye to the Hardware Husk.
- Draw a gold waypoint vector when a diagnostic alert has been selected.
- Fade/ring brightness reflects sector stability.
- Packet velocity is represented as a moving pulse on the ring.

## Waypoint Ghost

When a map alert is selected, the objective marker becomes the waypoint ghost.

Future world rendering can expand this into a wireframe path in the overworld. For now, the map marker and compass vector carry the behavior.

## Service Tickets

Sidequests are tickets generated by system anomalies.

Ticket types:

- Optimization: simple repair or NPC handshake.
- Unknown Code: exploration or real-world relic discovery.
- Corruption: Threadfall, de-rendering, Security Daemon activity.
- Root Quest: main story or Hardware Husk progression.

Ticket notifications should:

- Flicker/glitch in when entering a tile with an unresolved anomaly.
- Show title, ticket ID, type, and short system description.
- Set the selected tile as a waypoint when clicked from the map.

## Narrative Rule

Completing tickets should visibly stabilize the world:

- Greener grass.
- Less repetitive NPC dialogue.
- Fewer Threadfall alerts.
- Mirror Admin aggression can soften as Skye proves repairs preserve life instead of corrupting it.
