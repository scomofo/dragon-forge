# Dragon Forge Vertical Slice Plan

## Validation Question

Does a player, starting from a short cold open with no prior instruction, experience the Dragon Forge fantasy of bonding with a dragon in a fraying simulation within 3-5 minutes, and can the team build one representative hatch -> map -> battle -> reward loop at shippable direction quality?

## Scope

- **Start**: Three-beat cold open establishing Rendered World failure, Felix's signal, Skye's resident/operator status, one Root Egg, and the Bulkhead locked.
- **Challenge**: Hatch the Root Wyrmling, enter Village Edge, and fight one detached training protocol using the five-phase battle language.
- **Resolution**: Clear the node, gain Data Scraps, and return to the Forge with the loop complete.

## Systems Exercised

- Dragon Forge Hub: station prompt, Felix framing, Bulkhead progression.
- Hatchery: first hatch result, dragon identity, immediate companion framing.
- Campaign Map: one authored HAZARD node and a visible map state.
- Battle Engine: INIT/TELEGRAPH/IMPACT/RECOIL/RESOLUTION loop language in condensed form.
- Dragon Progression: visible dragon HP/name; no long-term XP curve in this slice.
- Economy: Data Scrap reward display after commit-style resolution.
- Input Router / UX: button-driven confirm/defend flow with no hover-only interaction.
- Save / Persistence: represented as clean state transitions; disk save is out of slice scope.

## Quality Bar

Placeholder presentation is acceptable, but flow language, UI hierarchy, button focus, and result clarity must be representative. This slice is reference only and must not be refactored into production.

## Success Criteria

- A player can complete the loop without developer guidance.
- First meaningful action occurs within 60 seconds.
- The player can explain why they hatched the dragon and why they entered the map node.
- Battle intent and available action are legible during TELEGRAPH.
- Reward resolution is unambiguous.
- The full playthrough takes 3-5 minutes.

## Hard Limit

If the demonstrable loop is not playable after three focused build days, stop and reassess scope or architecture assumptions.
