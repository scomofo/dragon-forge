## Implementation Notes

Add creation helpers for Hatchery core dragons, Fusion children, and Singularity story grants. Helpers should set canonical base stat fields, validate identity/element, and return named results. Do not implement Hatchery roll logic, Fusion formulas, or Singularity boss settlement here.

## Out of Scope

- Hatchery pull RNG and pity: Hatchery epic.
- Fusion formula and parent mutation: Fusion epic.
- Mirror Admin defeat and Void grant trigger: Singularity epic.

## QA Test Cases

- **AC-1**: Hatchery creation helper
  - Given: a staged save without a Fire dragon
  - When: Hatchery creation helper creates Fire
  - Then: a canonical Fire `DragonRecord` is staged with level 1, XP 0, canonical stats, and source metadata in the result
  - Edge cases: duplicate element routes to duplicate handling, not creation
- **AC-2**: Fusion child helper
  - Given: inherited base stats and `is_elder` from Fusion
  - When: Fusion creation helper is called
  - Then: the child uses supplied base stats, level 1, XP 0, shiny false, battle charges 0
  - Edge cases: helper rejects Void as a Fusion output
- **AC-3**: Void story grant helper
  - Given: normal roster capacity is full
  - When: Singularity helper grants `void_dragon`
  - Then: Void is placed in story roster, shiny is false, and stats match AC-DP06a/AC-DP66a
  - Edge cases: repeated grant is idempotent or rejected with named result
- **AC-4**: shiny immutability and missing duplicate target
  - Given: a hatched non-shiny dragon and a duplicate XP award for a missing element
  - When: public APIs attempt to set shiny or apply duplicate XP
  - Then: shiny remains unchanged and missing duplicate XP logs/discards per AC-DP76/AC-DP94
  - Edge cases: no null-reference crash
