## Important Architectural Rules

Do not violate these:

- Durable state changes go through Save / Persistence transactions.
- Battle, Campaign Map, Hatchery, Fusion, UI, and Presentation must not directly mutate durable save state.
- `dragon_id: StringName` is canonical dragon identity.
- `stage` is derived from level and must not be persisted.
- XP mutation belongs to Dragon Progression.
- Progression events are accumulated during staged mutation but emitted only after save commit success.
- Event payloads must be named Resources/classes, not anonymous Dictionaries.
- Battle runtime must be scene-local/runtime-only and must not open save transactions.
- Economy Ledger owns Scrap read/spend/reward boundaries.
- Battle animation lookup must go through `BattleAnimationManifest` + `MoveDefinition` + `BattleDefinition`, not hardcoded sprite branches.
